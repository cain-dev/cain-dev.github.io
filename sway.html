Arch Linux - Setup com Sway

Introdução

Sway é um compositor de Wayland e um substituto drop-in para o gerenciador de janelas i3 para X11. Ele funciona com sua configuração i3 existente e oferece suporte à maioria dos recursos do i3, além de alguns extras.
O Sway permite que você organize as janelas do aplicativo de forma lógica, em vez de espacialmente. As janelas são organizadas em uma grade por padrão, o que maximiza a eficiência da tela e pode ser rapidamente manipulado usando apenas o teclado.

Instalação

1 - Instalar os componentes

> sudo pacman -S sway alacritty waybar wofi xorg-xwayland qt5-wayland glfw-wayland ttf-font-awesome

2 - Copiar o arquivo template do Sway para a pasta de usuário

logue no seu usuário (não root) e entre no diretório de configurações de usuário (.config)

> cd ~/.config (se o diretório não for encontrado, crie ele executando o comando > mkdir .config)

Já dentro do diretório .config, crie o diretório do Sway com o comando > mkdir sway

Mova para o diretório do Sway com o comando > cd sway

Copie o arquivo de template do Sway (/etc/sway/config) para o diretório atual (.) com o comando > cp /etc/sway/config .

A configuração padrão de comandos do Sway é Configuração Padrão é:
Recarregar Sway: Super+Shift+c (não fechará programas que estejam rodando)
Sair do Sway: Super+Shift+e
Abrir Terminal: Super+Return
Abrir lançador: Super+d

3 - Editando o arquivo de configuração do Sway no diretório do usuário (/home/seuusuario/.config/sway/config)

Aqui vão algumas dicas de configurações para o seu Sway, você pode editar o arquivo de configuração com o nano

Acrescentar gaps e remover título:
Edite o .config/sway/config e mude os valores para

# A 10-pixel border around every window.
gaps inner 10
# Removes the title bar of each window.
default_border pixel 3

Utilizar a Waybar no Sway:
Edite o .config/sway/config e substitua o conteúdo de bar {} ao final do arquivo por

bar {
swaybar_command waybar
}

Habilitar o Wofi como drun (com ícones) no Sway:
Edite o .config/sway/config e substitua o conteúdo de

set $menu wofi --show=drun --lines=5 --prompt=""

por

set $menu wofi --show=drun -I

Mude o papel de parede:
Edite o .config/sway/config na linha output

output * bg /diretório/nomedaimagem.png fill

Carregar teclado ABNT2:
Edite o .config/sway/config e insira ao final do arquivo

input * {
xkb_layout "br"
xkb_variant "abnt2"
}

Adicionar habilitar o uso do touchpad em notebooks:
Edite o .config/sway/config e insira ao final do arquivo

input type:touchpad {
tap enabled
natural_scroll enabled
}

Mudar tecla mod para Alt:
Edite o .config/sway/config e altere de Mod4 para Mod1

# Logo key. Use Mod1 for Alt.
set $mod Mod1

Mudar comando de fechar janela para Mod+q:
Edite o .config/sway/config e altere o a linha abaixo de #Kill focused window para

bindsym $mod+q kill

4 - Copiar o arquivo de configuração padrão do Alacrity para o diretório .config do usuário

Acesse o diretório .config com > cd .config

Crie o diretório do Alacritty com > mkdir alacritty

Acesse o diretório do Alacritty com > cd alacritty

Copie o arquivo de template do Alacritty (/usr/share/doc/alacritty/example/alacritty.yml) para o diretório atual (.) com o comando > cp /usr/share/doc/alacritty/example/alacritty.yml .

5 - Editando o arquivo de configuração do Alacritty no diretório do usuário (/home/seuusuario/.config/alacritty/alacritty.yml)

Aqui vão algumas dicas de configurações para o seu Alacritty, você pode editar o arquivo de configuração com o nano

Alacritty transparente:
Edite o .config/alacritty/alacritty.yml e delete o "#" para habilitar a linha e mude o valor de 1.0 para a sua opacidade desejada (0.5 por exemplo)
# O valor de opacidade da janela é um número de ponto flutuante de `0.0` a` 1.0`.
# O valor `0.0` é completamente transparente e` 1.0` é opaco.

background_opacity: 0.5

mude a fonte do Alacritty:
Edite o .config/alacritty/alacritty.yml e em # Font configuration delete o "#" e habilite as linhas #font:, #normal:, #family: e #size:
na minha máquina eu utilizo family: SourceCodePro ( >sudo pacman -S adobe-source-code-pro-fonts para instalar) e size: 16

6 - Copiar o arquivo de configuração padrão do Waybar para o diretório .config do usuário

Acesse o diretório .config com > cd .config

Crie o diretório do Waybar com > mkdir waybar

Acesse o diretório do Waybar com > cd waybar

Copie o arquivo de template do Waybar (/etc/xdg/waybar/config) para o diretório atual (.) com o comando > cp /etc/xdg/waybar/config .

7 - Habilitar som e compartilhamento de tela

Para instalar o Pipewire:
pacman -S pipewire (também instale o libpipewire02 caso utilize o Chromium)

Para o Firefox:
Em /etc/environment adicione: MOZ_ENABLE_WAYLAND=1
para verificar execute > printenv

Para o Chromium:
Em /home/meuusuario/.config/chromium-flags.conf adicione:
--enable-features=UseOzonePlatform
--enable-gpu
--ozone-platform=wayland

Para setar as variáveis de ambiente para todos os usuários:
Em /etc/environment adicione:
XDG_SESSION_TYPE=wayland
XDG_CURRENT_DESKTOP=sway
para verificar execute > printenv

Para compartilhamento de tela, instale xdg-desktop-portal-wlr com:
pacman -S xdg-desktop-portal-wlr

Assim que for iniciar uma transmissão:
Do terminal, execute > /usr/lib/xdg-desktop-portal -r & /usr/lib/xdg-desktop-portal-wlr
Isso executará como um trabalho em segundo plano, portanto, não feche o terminal.
Abra seu navegador.
