Interfaces, Ambientes Gráficos e Gerenciadores de Janelas e Display

Introdução

Diferente do Windows e MacOS, os Linux e BSD's não se limitam a uma única interface, existem opções para sua tela de login, opções de ambientes gráficos completos ou mesmo opção para quem quer apenas um gerenciador de janelas. Aqui vou listar somente uma parte dentre aqueles mais comuns e com maior portabilidade, existem diveeersas outras opções além dessas.

Tudo é uma questão de capacidades vs recursos, e não de estilo!

Usuários leigos tendem a julgar todo um ambiente gráfico baseando-se na primeira vista desse ambiente, acontece que especialmente no caso de interfaces para Linux e BSD's o tema padrão chega quase a ser irrelevante em termos de visual e estilo pois as interfaces aqui não são apenas um estilo visual, uma estética, são parâmetros de mecanismos, objetos e recursos, são feitos visando as possibilidades de customização em relação a performance e não delimitam (muito) o estilo final. O ponto é: Não julgue a "beleza" da interface, pois ela é irrelevante já que presume-se que você vai customizá-la. Julgue as possibilidades e a relação dessas possibilidades com a exigência em processamento gráfico e uso de memória ram.

Ambientes Gráficos

Um ambiente de área de trabalho (desktop environment ou DE) reúne uma variedade de componentes para fornecer elementos de interface gráfica do usuário comum, como ícones, barras de ferramentas, papéis de parede e widgets de área de trabalho. Além disso, a maioria dos ambientes de desktop inclui um conjunto de aplicativos e utilitários integrados. Mais importante ainda, os ambientes de desktop fornecem seu próprio gerenciador de janelas, que pode, entretanto, ser substituído por outro compatível. Os Desktop Environments visam te dar uma experiência de uso completa, que vai da barra de tarefas até mesmo ao gerenciador de arquivos.

Mais uma vez: Essas imagens demonstram apenas o estado inicial desses ambientes. Todos eles são bastante customizáveis e alguns até ao ponto de ficarem irreconhecíveis, não julgue a beleza do ambiente se baseando apenas no tema padrão e no seu estado inicial: Instale, teste e customize antes de escolher ou julgar mal algum deles.

Gerenciador de Janelas

Um gerenciador de janelas (ou Window Manager) é um software de sistema que controla o posicionamento e a aparência das janelas em um sistema de janelas em uma interface gráfica de usuário (GUI). Ele pode fazer parte de um ambiente de área de trabalho (DE) ou ser usado de forma independente.

Display Managers / Login Managers

Um gerenciador de exibição, ou gerenciador de login, é uma interface gráfica exibida no final do processo de inicialização, em resumo, é a tela que aparece para você selecionar seu usuário e senha.

Certamente não para por aqui...

Minha intenção com essa página aqui não foi apresentar todas as possibilidades em ambientes gráficos mas sim demonstrar pra você que eles existem, tirar da mente do usuário final de que a interface do sistema é uma representação estática de como o sistema funciona. Cada interface tem seus objetivos, requisitos, possibilidades e uma vez que você entenda que você não precisa instalar o Linux Mint para utilizar a interface Cinnamon, ou mesmo instalar o Cinnamon para ter um estilo parecido com o do Windows você passa a entender um pouco da razão das pessoas instalarem um sistema Linux ou BSD. Eu utilizo o Cinnamon no meu computador pessoal devido a scripts de integração com a interface, utilizo IceWM e LXQt em máquina de baixíssima capacidade gráfica, utilizo o KDE quando quero um máximo de personalização... O ponto é para que você não fique preso a uma única interface ou uma única maneira de interagir com o seu sistema, escolha o seu sistema baseado no que ele pode oferecer e escolha a sua interface baseado em como você quer interagir com o sistema (especialmente considerando os recursos desse sistema).

Aqui não é como aquele caso que o usuário não pode atualizar o Windows 7 pro Windows 10 pois o computador não suporta a interface...
