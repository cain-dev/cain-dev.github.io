Instalação do Arch Linux

Instalação Alvo

Como o Arch Linux é um sistema bastante configurável, toda instalação tem um alvo de como ser finalizada. A instalação alvo dessa documentação trata-se de um sistema em UEFI com systemd-boot, com raiz em EXT4 e iwd+resolved OU NetworkManager+resolved. Portanto, se você tem um computador que funciona somente com BIOS ou queira outro bootloader como o GRUB para utilizar dualboot Não siga essa documentação para particionamento e instalação de bootloader.

⚠️Alerta: Necessário um mínimo de experiência prévia com sistemas Unixlike

// 1 - Configurar teclado

Antes de digitar qualquer comando é necessário que o teclado esteja configurado corretamente no padrão ABNT2, para isso utilize o comando:

loadkeys br-abnt2

// 1B - Mudar a fonte da padrão do terminal (opcional)

Eu frequentemente tenho que instalar o ArchLinux num laptop e a fonte fica pequena demais, para alterar para uma fonte mais agradável eu utilizo o comando:

setfont ter-124b

// 2 - Configurar rede

para cabo Ethernet

Caso você utilize um cabo de rede basta verificar aquisição de um endereço IP válido e conexão com a rede através dos comandos:

ip address show

E

ping google.com

Se a conexão com o Google não teve falhas você pode prosseguir para a edição de mirrors

para Wifi

Caso você esteja utilizando uma conexão sem fio é necessário utilizar o iwd para fazer a conexão, para iniciá-lo execute:

iwctl

Agora é necessário listar os dispositivos de conexão sem fio que estão disponíveis, para isso utilize o comando:

device list

Uma vez que você sabe qual é o nome do dispositivo responsável pela conexão sem fio no seu computador utilize os comandos abaixo para listar as redes disponíveis substituindo device para o nome do seu dispositivo:

station device scan station device get-networks

Depois que você achou a rede Wifi basta utilizar o comando abaixo para se conectar substituindo SSID pelo nome da sua rede:

station device connect SSID

Assim que se conectar você pode sair do programa e continuar para a instalação através do comando:

exit

Verifique a aquisição de um endereço IP válido e conexão com a rede através dos comandos

ip addr show

E

ping google.com

Se a conexão com o Google não teve falhas você pode prosseguir para a edição de mirrors

// 3 - Editar o arquivo de mirrors

Existem diversam maneiras de editar o arquivo de mirrors e habilitar apenas os servidores brasileiros, mas a que geralmente uso é através do comando de substituição do vim, você pode abrir o arquivo de mirrors no vim com o seguinte comando:

vim /etc/pacman.d/mirrorlist

Com o vim aberto, escreva o comando abaixo para comentar e desabilitar todos os servidores:

:%s/Server/#Server/g

Uma vez que todos os servidores foram comentados, utilize as setas do teclado e o delete para apagar o # dos servidores brasileiros. Quando finalizar utilize o comando abaixo para salvar e sair:

:wq

// 4 - Atualizar pacotes

Antes de interagir qualquer outro programa do sistema é interessante atualizar os pacotes com o seguinte comando:

pacman -Syy

// 5 - Particionamento

Antes de começar qualquer particionamento é importante ter certeza de qual disco vamos alterar, para listar os discos da máquina utilize o comando:

fdisk -l

Uma vez que você tem a certeza de qual disco vai alterar e instalar o sistema utilize o comando abaixo para iniciar o particionamento, lembre-se de substituir o x em sdx pela letra do seu disco:

gdisk /dev/sdx

Já com o gdisk aberto no disco selecionado criaremos uma nova tabela de partição em GPT com o comando:

o

Partição EFI

Antes de mais nada é importante criarmos a partição EFI, o comando para criar uma nova partição é:

n

O sistema irá exibir a seguinte linha para que você escolha o número da partição:

partition number: default

Apenas tecle Enter

Logo em seguida o sistema irá exibir a seguinte linha para que você escolha o início da partição:

first sector: default

Apenas tecle Enter

Logo em seguida o sistema irá exibir a seguinte linha para que você escolha o término da partição e delimite o seu tamanho, por desencargo de consciência utilizaremos 512MiB, sendo assim deveremos utilizar o comando +512M e teclar Enter de maneira que o diálogo no terminal ficará da seguinte forma:

last sector: +512M

Logo em seguida o sistema irá exibir a seguinte linha para que você escolha o tipo da partição, utilizaremos o tipo ESP, sendo assim deveremos utilizar o comando ef00 e teclar Enter de maneira que o diálogo no terminal ficará da seguinte forma:

Hex code or GUID (L to show codes, Enter = 8300): ef00

Partição de swap (opcional)

Digite o comando abaixo para criar uma nova partição:

n

O sistema irá exibir a seguinte linha para que você escolha o número da partição:

partition number: default

Apenas tecle Enter

Logo em seguida o sistema irá exibir a seguinte linha para que você escolha o início da partição:

first sector: default

Apenas tecle Enter

Logo em seguida o sistema irá exibir a seguinte linha para que você escolha o término da partição e delimite o seu tamanho, o tamanho recomendado dessa partição varia conforme o quanto de memória ram tem no seu computador e se você vai ou não utilizar hibernação. Verifique a tabela abaixo e substitua os valores de acordo com o seu caso, no meu caso utilizarei 3G portanto devo utilizar o comando +3G e teclar Enter de maneira que o diálogo no terminal ficará da seguinte forma:

last sector: +3G

Logo em seguida o sistema irá exibir a seguinte linha para que você escolha o tipo da partição, para a partição swap deveremos utilizar o comando 8200 e teclar Enter de maneira que o diálogo no terminal ficará da seguinte forma:

Hex code or GUID (L to show codes, Enter = 8300): 8200

Para / (ou para /home)

Digite o comando abaixo para criar uma nova partição:

n

O sistema irá exibir a seguinte linha para que você escolha o número da partição:

partition number: default

Apenas tecle Enter

Logo em seguida o sistema irá exibir a seguinte linha para que você escolha o início da partição:

first sector: default

Apenas tecle Enter

Logo em seguida o sistema irá exibir a seguinte linha para que você escolha o término da partição e delimite o seu tamanho, caso que você queira que essa partição vá até o fim do disco apenas tecle Enter, caso você queira que ela tenha 100G você deve utilizar o comando +100G e teclar Enter de maneira que o diálogo no terminal ficará da seguinte forma:

last sector: +100G

Logo em seguida o sistema irá exibir a seguinte linha para que você escolha o tipo da partição, utilizaremos o tipo 8300, seja para / ou /home, o tipo 8300 já é o padrão portanto basta teclar Enter no diálogo abaixo:

Hex code or GUID (L to show codes, Enter = 8300):

Gravar e sair

Para gravar as alterações da tabela de partição e sair do programa utilize o comando:

w

// 6 - Verificar partições

Antes de prosseguir com a formatação e montagem das partições é importante ter certeza de que a tabela está correta, para listar as partições de disco utilize o comando abaixo:

lsblk

// 7 - Formatar partições

Uma vez que a tabela de partição já foi terminada é importante formatarmos cada partição para o seu devido formato de arquivos, a primeira partição a ser formatada será a partição ESP, ela deve ser formatada em formato FAT32. Considerando que a sua partição ESP é a /dev/sda1 utilize o comando abaixo para formatar (OU substitua /dev/sda1 para o devido endereço da sua tabela de partições):

mkfs.fat -F32 /dev/sda1

(considerando o seu disco como /dev/sda e 1 como o número da partição efi)

A segunda partição a ser formatada será a partição swap caso você a tenha feito, ela deve ser formatada em formato swap. Considerando que a sua partição swap é a /dev/sda2 utilize o comando abaixo para formatar (OU substitua /dev/sda2 para o devido endereço da sua tabela de partições):

mkswap /dev/sda2

(caso use partição swap e considerando o seu disco como /dev/sda e 2 como o número da partição swap)

Por fim formataremos a partição raiz do sistema /, ela deve ser formatada em formato EXT4. Considerando que a sua partição / é a /dev/sda3 utilize o comando abaixo para formatar (OU substitua /dev/sda3 para o devido endereço da sua tabela de partições):

mkfs.ext4 /dev/sda3

(considerando o seu disco como /dev/sda e 3 como o número da partição raiz, caso você tenha feito uma partição /home ela deve ser formatada com o mesmo comando mkfs.ext4 /dev/sdax substituindo o x pela devida numeração na sua tabela de partições)

// 8 - Montar partições

Uma vez que a tabela de partições está pronta e as partições já estão com os seus devidos formatos de arquivo agora falta montá-las nos seus devidos endereços para proseguirmos com a instalação. Para montar a partição raíz do sistema (/) utilize o comando:

mount /dev/sdxy /mnt

(altere o x para a letra do disco e o y para o número da partição /)

Se e somente se você tenha feito uma partição separada para o diretório /home você deve criar o diretório /home com o comando:

mkdir /mnt/home

E montá-la partição com o comando:

mount /dev/sdxy /mnt/home

(altere o x para a letra do disco e o y para o número da partição /home)

Se e somente se você tenha feito uma partição dedicada para swap você deve ativá-la com o comando:

swapon /dev/sdxy

(altere o x para a letra do disco e o y para a partição swap)

Agora precisamos criar o diretório de inicialização, para isso utilize o comando:

mkdir /mnt/boot

E monte-o com o comando:

mount /dev/sdxy /mnt/boot

(altere o x para a letra do disco e o y para o número da partição ESP)

// 9 - Instalar sistema

Uma vez que todos os diretórios foram criados e montados só falta agora instalar o sistema através do pacstrap, no meu caso utilizarei uma instalação mínima com o kernel linux padrão, wifi com o iwd, com o nano para editor de texto, updates de microcódigo com o pacote amd-ucode e elevação de privilégios com o sudo. Então essa vai ser o MEU comando de instalação:

pacstrap /mnt base linux linux-firmware iwd nano amd-ucode sudo

Porém:

  • Caso você queira utilizar um kernel lts você pode substituir o pacote linux por linux-lts;

  • Caso você utilize cabo de rede ao invés de wifi você deve substituir o pacote iwd por networkmanager;

  • Caso você utilize processadores Intel ao invés de AMD você deve substituir o pacote amd-ucode por intel-ucode;

  • Você também pode optar por não ter elevação de privilégios através do sudo ou mesmo utilizar o doas ao invés dele;

// 10 - Gerar FSTAB

Já com o sistema instalado podemos gerar o arquivo da tabela do sistema de arquivos com o comando:

genfstab -U /mnt >> /mnt/etc/fstab

Para verificar o arquivo da tabela do sistema de arquivos utilize o comando:

cat /mnt/etc/fstab

//11 - CHROOT para a instalação

Com a tabela pronta e verificada acessaremos a instalação com o comando:

arch-chroot /mnt

//11B - Criar swapfile (OPCIONAL)

Se e somente se você não tenha feito partição dedicada para swap mas ainda deseje utilizar swap através de arquivo swap, aloque o espaço de arquivo swap com o comando:

fallocate -l 3GB /swapfile

(Considerando que você queira utilizar 3GiB de swap, altere o 3 para a quantia de swap desejada)

Mude a permissão do swapfile com o comando:

chmod 600 /swapfile

Configure o arquivo como área para troca de swap com o comando:

mkswap /swapfile

Ative a swap com o comando:

swapon /swapfile

Agora devemos adicionar esse arquivo como parte da tabela de sistema de arquivos, para acessar o arquivo da tabela de sistema de arquivos utilize o comando:

nano /etc/fstab

E ao final do arquivo adicione a linha:

/swapfile none swap defaults 0 0

//12 - Configurar o sistema instalado para teclado ABNT2

echo KEYMAP=br-abnt2 >> /etc/vconsole.conf

//13 - Setar timezone e ajustar relógio do sistema

ln -sf /usr/share/zoneinfo/America/Sao_Paulo /etc/localtime hwclock --systohc

//14 - Setar locale

Utilize o nano para acessar o arquivo de configuração de localização com o comando:

nano /etc/locale.gen

Habilite a localização pt_BR.UTF-8 UTF-8 descomentando a linha deletando o # antes dele e após isso gere os arquivos de localização com o comando:

locale-gen

Para setar a configuração padrão de localização do sistema utilize o comando:

echo "LANG=pt_BR.UTF-8" >> /etc/locale.conf

//15 - Configurar o arquivo de hosts da instalação

Utilize o comando abaixo para selecionar o nome do host substituindo meuhostname para o nome de host desejado:

echo meuhostname >> /etc/hostname

Logo em seguida edite o arquivo de hosts com o nano com o comando:

nano /etc/hosts

Nesse arquivo adicione as linhas abaixo meuhostname para o nome de host desejado:

127.0.0.1 localhost ::1 localhost 127.0.1.1 meuhostname.localdomain meuhostname

//16 - Adicionar senha do administrador

Para definir a senha de administrador utilize o comando:

passwd

//17 - Instalar bootloader

Para instalar o bootloader utilize o comando:

bootctl --path=/boot install

Uma vez que o bootloader foi instalado utilize o nano para editar as configurações do bootloader com o comando:

nano /boot/loader/loader.conf

Adicione a linha:

default arch.conf

Isso fará com que o arquivo padrão a ser carregado pelo sistema seja o arch.conf que ainda não foi criado, então utilizaremos o nano para criar esse arquivo com o comando:

nano /boot/loader/entries/arch.conf

Nesse arquivo adicione as linhas:

title Archlinux linux /vmlinuz-linux initrd /amd-ucode.img initrd /initramfs-linux.img options root=/dev/sdxy rw

Observações:

  • Caso você utilize processadores Intel ao invés de AMD você deve substituir o amd-ucode.img por intel-ucode.img;

  • Em options root=/dev/sdxy rw altere o x para a letra do disco e o y para o número da partição /;

//18 - Habilitar e configurar serviços de rede

Para Wifi

Caso você tenha optado por utilizar Wifi habilite os serviços iwd e systemd-resolved com os comandos:

systemctl enable iwd.service

systemctl enable systemd-resolved.service

Agora devemos editar o arquivo de configuração do iwd, para acessar o arquivo pelo nano utilize o comando:

nano /etc/iwd/main.conf

Desde a versão 0.19, o iwd pode atribuir endereços IP e configurar rotas usando um cliente DHCP integrado, para habilitar essa configuração adicione as linhas:

[General]
EnableNetworkConfiguration=true

Para habilitar o systemd-resolved adicione as linhas:

[Network]
NameResolvingService=systemd

Desde a versão 1.10, o iwd suporta IPv6, mas está desabilitado por padrão. Caso queira habilitá-lo ainda nas opções de [Network] adicione a linha:

EnableIPv6=true

Para cabo Ethernet

Caso você tenha optado por utilizar cabo Ethernet habilite os serviços NetworkManager e systemd-resolved com os comandos:

systemctl enable NetworkManager.service systemctl enable systemd-resolved.service

NOTA:

Essa configuração foi escolhida focando por mais se encaixar no meu perfil de uso e não necessariamente em facilidade de uso. Utilizo o iwd por basicamente ser um sucessor do wpa_supplicant e já ter cliente DHCP integrado, sendo assim basta a integração com o resolved. E no caso do NetworkManager por também ter cliente DHCP integrado, altíssima compatibilidade com outros pacotes e ser o go to option da maioria das distribuições.

//19 - Criar usuário e habilitar elevação de privilégios com sudo

Para adicionar um usuário que possa utilizar sudo utilize o comando abaixo substituindo o seunomedeusuário pelo nome de usuário que você deseja:

useradd -mG wheel seunomedeusuário

Para configurar a senha da conta desse usuário use o comando abaixo substituindo o seunomedeusuário pelo seu nome de usuário:

passwd seunomedeusuário

Para editar o arquivo sudoers com o nano utilize o comando abaixo:

sudo env EDITOR=nano visudo

Habilite a elevação de privilégio para o grupo wheel descomentando a linha deletando o # antes dele em:

%wheel ALL=(ALL) ALL

//20 - Finalizando a instalação

Primeiramente precisamos sair do ambiente chroot com o comando:

exit

Se e somente se você ativou swap desligue-a com o comando:

swapoff

Agora desmontaremos as partições restantes com o comando:

umount -a

Para reiniciar o computador e concluir a instalação execute o comando:

reboot

Lembre-se de remover a mídia de instalação assim que o computador desligar

//Retoques pós-instalação

Ativando o TRIM:

Caso você utilize SSD's você pode otimizá-los ativando o serviço TRIM com o comando:

systemctl enable fstrim.timer

Habilite cor no pacman:

Para ativar cor no pacman basta acessar o arquivo de configuração do pacman com o nano com o comando:

nano /etc/pacman.conf

Descomente a linha deletando o # antes de Color

Habilite noatime:

Acesse o fstab com o nano com o comando:

nano /etc/fstab

Mude de:

UUID=XXXXXXXXXXXXXXXXXXXX / ext4 relatime,errors=remount-ro 0 1

Para:

UUID=XXXXXXXXXXXXXXXXXXXX / ext4 noatime,errors=remount-ro 0 1

E em seguida salve o arquivo

//Instalando o Cinnamon

Utilize o comando:

pacman -S xorg cinnamon lightdm-gtk-greeter-settings gnome-terminal

Habilite o serviço do LightDM com o comando:

systemctl enable lightdm

Habilitar o Folder Color no Nemo:

Para facilitar a instalação e manutenção dos pacotes do aur utilizaremos o yay. Logue na sua conta de usuário (não na do root) e instale as dependências necessárias com o comando:

sudo pacman -S git base-devel nemo-python

Clone o diretório do github com o comando:

git clone https://aur.archlinux.org/yay.git

Acesse o diretório do yay com o comando:

cd yay

Crie o pacote e instale o yay com o comando:

makepkg -si

Após o fim da instalação utilize o yay para instalar os pacotes necessários com o comando:

yay -S mint-x-icons mint-y-icons folder-color-switcher

Para fazer com que as mudanças sejam carregadas utilize o comando abaixo para fechar o Nemo e depois abra novamente:

nemo -q

Habilitar extração/compressão no menu de contexto do Nemo:

Basta executar o seguinte comando:

sudo pacman -S nemo-fileroller
